HTML

  • HTML = Hypertext Markup Language
  • HTML gives structure to our content.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS = Cascading Style Sheets
  • CSS gives style to our content.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commit all files in the working branch

JavaScript

  • JavaScript handles logic and allows users to interact with a website.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript,a file is read from top to bottom, which means control flow wise, everything will be executed from top to bottom.
  • There are 2 requirements for a JavaScript function to run properly: we need to 1) define the function and 2) call the function.
  • There are multiple ways to define a function but one of the most common is a function declaration, which makes use of the function keyword.
  • If our function's pre-defined conditions do not match at any point, we will get an error warning.
  • Errors can be caused by (among other things) typos, changes to variable names and control flow mismatch.
  • In terms of control flow mismatch, as an example, if we attempt to use a variable before it is define, that will trigger an error in our function.
